# Security Policy

## Threat Model

Fluxo is a minimal local browser extension wallet. It assumes the browser profile and device are trusted while the wallet is unlocked. It does not protect against malware, malicious browser extensions, compromised operating systems, keyloggers, screen capture, memory inspection, or browser profile theft.

This project currently avoids high-risk wallet surfaces:

- no RPC writes
- no transaction broadcasting
- no content script injection
- no dapp provider injection
- no cloud sync
- no background service that keeps keys unlocked

## Secret Handling

- New wallets use a BIP39 seed phrase generated by the Go WASM wallet core.
- The seed phrase is encrypted before storage by the Go WASM vault layer.
- Vault v3 uses XChaCha20-Poly1305 with a 24-byte nonce.
- The encryption key is derived with Argon2id using 256 MiB memory, 4 passes, `p=1`, a 32-byte salt, and a 32-byte key.
- Vault metadata is authenticated as AEAD additional data.
- KDF downgrade below the v3 defaults is rejected.
- Only encrypted vault data is saved in `chrome.storage.local`.
- JavaScript stores only the encrypted vault, address, active chain, and short-lived session ID.
- JavaScript receives the seed phrase only once during wallet creation for user backup.
- The seed phrase and derived private keys stay behind the Go session boundary after backup.
- Closing or hiding the popup locks the active session best-effort; Go sessions also have a 5-minute TTL.
- Legacy v1 vaults are supported only for unlock-and-migrate to legacy private-key v2.

Browser JavaScript and Go WASM memory cannot provide hardened zeroization guarantees. This design is hardened against offline brute force and vault tampering under the stated threat model, but it is not a hardware wallet and it is not "unhackable." For high-value funds, prefer a hardware wallet.

## Vault v3 Format

The persisted object has this high-level shape:

```json
{
  "header": {
    "version": 3,
    "cipher": "XCHACHA20-POLY1305",
    "kdf": "ARGON2ID",
    "kind": "hd-mnemonic",
    "kdfParams": {
      "memoryKiB": 262144,
      "passes": 4,
      "parallelism": 1,
      "saltBytes": 32,
      "keyBytes": 32
    },
    "address": "0x...",
    "createdAt": "..."
  },
  "salt": "...",
  "nonce": "...",
  "ciphertext": "..."
}
```

The `header` is authenticated as AEAD additional data. Changing the address, cipher, KDF parameters, nonce, or ciphertext causes unlock failure.

## Limitations

- `chrome.storage.local` is persistent extension storage, not a secret store.
- A compromised browser process, malicious extension with sufficient access, or local malware can still attack the wallet while unlocked.
- Password strength still matters. Argon2id slows offline guessing; it does not make weak passwords safe.
- Balance reads call configured public RPC endpoints with the active address and chain ID.
- No transaction broadcast, dapp provider, content script, or permission system exists in this scope.

## User Guidance

- Use a strong, unique vault password.
- Test with throwaway accounts first.
- Do not use this extension in a browser profile with untrusted extensions.
- Back up the encrypted vault and password separately.
- Store the seed phrase offline. Anyone with the seed phrase can control the account.
- Do not store meaningful funds until the code has been independently reviewed.

## Reporting Vulnerabilities

Please report vulnerabilities privately before public disclosure.
