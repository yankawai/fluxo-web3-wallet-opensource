# Security Policy

## Threat Model

`go-web3-wallet` is a minimal local browser extension wallet. It assumes the browser profile and device are trusted while the wallet is unlocked. It does not protect against malware, malicious browser extensions, compromised operating systems, keyloggers, screen capture, or browser profile theft.

This project currently avoids high-risk wallet surfaces:

- no automatic RPC calls
- no transaction broadcasting
- no content script injection
- no dapp provider injection
- no cloud sync
- no background service that keeps keys unlocked

## Secret Handling

- The private key is generated by the Go WASM wallet core.
- The key is encrypted before storage using AES-GCM.
- The AES key is derived from the vault password with PBKDF2-SHA256.
- Only encrypted vault data is saved in `chrome.storage.local`.
- The plaintext private key is kept only in popup memory while unlocked.
- Closing the popup clears the runtime memory for the unlocked session.

Browser JavaScript and Go WASM memory cannot provide hardened zeroization guarantees. For high-value funds, prefer a hardware wallet.

## User Guidance

- Use a strong, unique vault password.
- Test with throwaway accounts first.
- Do not use this extension in a browser profile with untrusted extensions.
- Back up the encrypted vault and password separately.
- Do not store meaningful funds until the code has been independently reviewed.

## Reporting Vulnerabilities

Please report vulnerabilities privately before public disclosure.
