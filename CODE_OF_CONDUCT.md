# Code of Conduct

This project expects professional, security-aware collaboration.

Do:

- discuss technical tradeoffs with evidence;
- disclose security concerns responsibly through `SECURITY.md`;
- avoid publishing secrets, exploit details, or sensitive wallet material;
- keep reviews focused on correctness, maintainability, and user safety.

Do not:

- harass contributors;
- spam issues or pull requests;
- pressure maintainers to make custody-grade claims without hardware, audits, and evidence;
- post private keys, seed phrases, passwords, or production vault data.

Maintainers may edit, hide, or remove content that puts users or contributors at risk.
